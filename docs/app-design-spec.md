# 애플리케이션 설계 사양: 로컬 LLM CLI 연동 및 멀티 에이전트 대화 관리

## 1. 개요 (Overview)
본 문서는 로컬에 설치된 `gemini-cli` 및 향후 추가될 LLM(Claude, OpenAI 등)의 입출력을 macOS 파이프 기능을 통해 연동하고, 사회자(사용자) 중심의 멀티 에이전트 대화 환경을 제공하는 웹 애플리케이션의 설계를 정의합니다.

## 2. 핵심 설계 원칙 (Core Design Principles)
- **Clean Architecture**: 핵심 도메인 로직(대화 중재, 컨텍스트 관리)과 외부 연동(CLI, Web)을 철저히 분리합니다.
- **Fail-Fast**: 프로세스 통신 또는 데이터 처리 중 발생하는 예외를 즉시 보고하여 안정성을 확보합니다.
- **Extensibility**: 인터페이스 기반 설계를 통해 새로운 LLM 엔진 추가 시 기존 코드 수정을 최소화합니다.

## 3. 기술 사양 (Technical Specifications)

### 3.1. LLM CLI 연동 및 통신
- **통신 기제**: macOS의 `subprocess.Popen`을 사용하여 로컬 CLI 프로세스를 실행하고, 표준 입출력(`stdin`, `stdout`)을 파이프로 연결하여 비동기 스트리밍 방식으로 데이터를 주고받습니다.
- **공통 인터페이스 (`ILLMConnector`)**:
    - `send_message(prompt: str, context: dict) -> Generator[str, None, None]`
    - 모든 LLM 엔진(Gemini, Claude 등)은 이 인터페이스를 준수하는 어댑터를 통해 구현됩니다.

### 3.2. 멀티 에이전트 대화 메커니즘
- **참여 주체**: 사용자(사회자 역할)와 하나 이상의 에이전트가 동시에 대화에 참여합니다.
- **사회자(Moderator) 제어**: 사용자가 웹 UI를 통해 발언 순서를 지정하거나, 에이전트 간 대화를 중재합니다.
- **대화 종료 규칙 (End-of-Turn)**:
    - 에이전트는 답변 완료 시 특정 시그널(예: `[DONE]` 토큰, `EOF`, 또는 스트림 종료)을 발생시킵니다.
    - 시스템은 이를 감지하여 제어권을 다시 사회자(사용자)에게 반환합니다.

### 3.3. 페르소나 및 데이터 관리
- **페르소나 (Persona)**: 에이전트의 이름과 시스템 지시문(System Prompt)을 저장하여 재사용할 수 있도록 관리합니다. 즐겨찾기(`is_favorite`) 기능을 통해 자주 사용하는 페르소나를 우선 관리합니다.
- **컨텍스트 및 이력 관리**:
    - 모든 메시지는 LLM 종류, 페르소나 ID, 대화 시각, 응답 시간 등의 메타데이터와 함께 기록됩니다.
    - **컨텍스트 윈도우 전략**: 각 LLM의 컨텍스트 한계를 고려하여, 과거 이력의 자동 요약(Summarization) 또는 슬라이딩 윈도우 기반의 컨텍스트 절삭 전략을 적용합니다.
    - 과거 채팅 내역은 Gemini 웹 인터페이스와 유사한 이력 조회 기능을 통해 제공됩니다.

## 4. 사용자 인터페이스 (UI/UX)
- **웹 기반 채팅창**: 실시간 스트리밍 답변을 지원하는 모던한 채팅 인터페이스.
- **사회자 컨트롤러**: 참여 에이전트 선택, 발언권 부여, 페르소나 설정 및 전환 기능 제공.
- **설정 및 저장**: 페르소나 라이브러리 관리 및 이전 대화 세션 로드 기능.

## 5. 데이터 스키마 (Data Schema) - 개요
- **Message**: `id`, `session_id`, `sender_type`, `content`, `metadata`, `created_at`
- **Persona**: `id`, `name`, `instruction`, `engine_type`, `updated_at`

## 6. 보안 및 예외 처리
- **프로세스 격리**: 각 CLI 프로세스는 독립적인 세션으로 관리됩니다.
- **비정상 종료 대응**: CLI 프로세스 충돌 시 자동 재시작 및 상태 복구 로직을 포함합니다.
